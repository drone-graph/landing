Graph memory as a truth standard.

Every other agent stack treats memory as a chat history with retrieval glue. Drone Graph separates memory into distinct graph spaces, each with its own truth standard, lifetime, and access rules.

Tasks

What is happening. Goals, atomic tasks, dependencies, ownership, retries, blockers. The execution structure of the mission. Lives only as long as the mission does.

Findings

What is being learned. Hypotheses, raw evidence, partial conclusions, weak signals, unresolved contradictions. Temporary by design. The Sleeper layer prunes stale findings; the Gatekeeper promotes verified ones.

Knowledge

What is known. Verified facts, validated decisions, durable conclusions. Every entry carries source, timestamp, confidence, provenance chain, verification status. Long-term memory — but structured.

Findings are not knowledge. Knowledge is not chat history. Treating them the same is how agent systems hallucinate themselves into incoherence.

A mission, end to end. Self-organizing AI for civilization-scale work.

A user submits a goal: Identify possible new planet candidates in this Milky Way observational dataset. Here is what happens.

  1. Intake

    The user's goal, dataset, and constraints enter the system.

  2. Routing

    Tasks are assigned to disposable drone instances based on capability heuristics.

  3. Execution

    Drones spawn, load the skills they need, do the work, emit findings, and terminate.

  4. Placement

    Deterministic gatekeepers decide: raw anomaly note → Findings Graph; validated candidate score → Knowledge Graph; unsupported speculation → quarantine.

  5. Alignment

    Drones are spawned to check whether the actual output satisfies the original user goal. If not, new tasks are generated and the loop continues.

  6. Return

    When the goal is satisfied, results are returned. The drones are gone. The Knowledge Graph remains.
