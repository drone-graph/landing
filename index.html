Other agent stacks treat memory as one flat bucket — chat history, a folder of .md notes, a vector store, whatever. Drone Graph has one shared substrate that outlives every drone, every mission, every restart. Five kinds of content live in it, each with its own lifecycle.

Persistence is the default, not a guarantee. Stale tools get uninstalled. Findings get summarized and compacted. Emergent gaps close when filled. The collective mind itself never goes away.

Skills

Runnable procedures any drone can load, download from the internet, or author from scratch. A skill written by one drone belongs to every future drone. Accumulates monotonically.

Tool Registry

Starts empty. Populated as drones install packages through the terminal. Tools unused within a window get uninstalled to save disk. Persistent with expiry.

Findings

Venv state, research notes, business progress, execution history. Loaded into memory only when needed, and summarized and compacted otherwise. Persistent, but not verbatim.

User Uploads

Files and context the human hands in. Held verbatim until the user removes them. The one place the swarm doesn't paraphrase you.

Gaps

The full surface of open work. Preset gaps (gap finding, memory management, research, testing) are never closed, only continually worked. Emergent gaps surface and close.

A drone is an executor, not a memory. Conflating the two is how every other agent stack hallucinates itself into incoherence.

Civilization-scale work. Zero employees.

A user submits a goal: Launch and operate a pharmaceutical company. Discover, develop, trial, and ship a new drug for Parkinson's. The swarm starts. It does not stop.

  1. Intake

    The user provides target indication, molecule libraries, corporate entity, budget, and jurisdiction. Everything lands in user uploads. The goal enters the collective mind as the first non-preset gap.

  2. Gap finding

    A drone on the gap-finding preset gap decomposes the goal: target validation, in-silico screening, preclinical work, trial design across phases, FDA filings, manufacturing scale-up, distribution, post-market surveillance. When a branch of the gap tree reaches roughly five layers deep, leaf gaps become executable.

  3. Drones spawn

    For each leaf gap, an ephemeral drone wakes. One runs compound-similarity searches. Another drafts an FDA pre-IND briefing. A third reconciles supplier invoices. They set their own reasoning depth and model tier — deep reasoning for trial design, fast tool-use for procurement. Same system prompt. No identity.

  4. Terminal execution

    Every drone acts through the terminal — the only real tool. API calls, database queries, submission portals, contract drafts, emails, payments. Each drone checks the signal protocol first: don't install what's already installed, don't open a file another drone has open.

  5. Findings deposited

    Binding-affinity results land in findings. A skill for draft FDA pre-IND briefing is written once and reused for every future drug. The tool registry grows as packages install. The drone dissolves.

  6. Preset gaps keep working

    Research scans literature for new indications. Testing runs continuous validation on the pipeline. Memory management summarizes decade-old trial data. Gap finding keeps decomposing as the company grows from one drug to ten.

  7. Human in the loop

    The hivemind pings the user only for what it can't do on its own: wet-lab execution, IRB signatures, patient-dosing authorization, wire transfers above threshold. The list is small by design. Everything digital is the hivemind's problem.

A pharmaceutical company with zero employees. No org chart. No quarterly reviews. The work organizes itself around the gaps. The collective mind remembers what the drones cannot.
